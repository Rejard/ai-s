Core State Engine

AIDL은 이름과 제어 구조가 같은 엔진입니다

외부적으로는 Adaptive Intelligence & DNA Logic, 내부적으로는 유전자 상태 체계 Active / Inactive / Deprecated / Lethal을 뜻합니다. 실제 런타임과 관리자 제어는 이 4상태를 기준으로 작동합니다.

A

Active

실시간 발현, 평가, 교배에 참여합니다.

I

Inactive

현재 세대에서는 휴면이지만 돌연변이로 재활성화될 수 있습니다.

D

Deprecated

완전 제거 대신 감쇠 상태로 남아 표현 강도가 낮아집니다.

L

Lethal

치명적 위험이 확인된 유전자로 발현에서 제외되고 자손에는 비활성으로 낮춰집니다.

AI-VEP Safety Gate

위험 변이는 sandbox 이전에 차단됩니다

AI-VEP는 변이 유전자의 위험 행동을 먼저 검사합니다. 현재 실제 구현은 4가지 대표적 위험 편향을 막고, 최근 3세대 holdout 붕괴가 보이면 lethal 스크리닝을 더 강화합니다.

BEAR_EXPANSION + knife-catching BUY bias
하락 확장기 무리한 저가매수 편향 차단
BEAR_SQUEEZE + breakout overcommit
하락 압축기 돌파 과몰입 차단
BULL_EXPANSION + late-trend chase bias
상승 확장기 후행 추격 편향 차단
BULL_SQUEEZE + premature SELL bias
상승 압축기 조기 매도 편향 차단

Admin-Only Operations

AIDL 운영 정보는 관리자 전용으로 다룹니다

후보군, 계보, 수리 이력, 유전자 상태 제어는 일반 회원용 기능이 아닙니다. 현재 운영면에서 중요한 것은 lineage 추적, accession 식별, repair telemetry, 수동 상태 override입니다.

Lineage
AISG-G12-7C41

parent_ids, ancestor_ids, innovation_ids를 기준으로 생존 계보를 추적합니다.

Repair Telemetry
self-heal audit trail

accession id, context_mask, regulatory_profile 누락을 runtime에서 수리하고 관리자에게 노출합니다.

Manual Override
A / I / D / L force set

관리자가 전략 유전자를 강제로 상태 전환하면 phenotype 재계산과 mutation_log 기록이 동반됩니다.

Archive View
culled lineage summary

archive reason과 mutation-event depth를 통해 탈락 후보의 흔적도 볼 수 있습니다.

[admin] genome AISG-G12-7C41 lineage depth = 14
[repair] context_mask missing -> restored with 4 regular climates
[override] gene breakout_17 state A -> L, event admin_state_override
[archive] CULLED_LOW_PERFORMANCE with mutation depth = 5
